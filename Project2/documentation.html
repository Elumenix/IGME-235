Process

Everything wen't pretty smoothly for the most part and followed my schedule pretty well, except for the documentation, which I only spottend on the rubric as I went to upload my project. I worked on all the javascript before I began touching CSS. After a lot of confusion I was able to access the api and from there it was very easy to put an image on the screen. It wasn't difficult to add the options in either. Local storage was much more difficult, and there were multiple points when it required me to change major sections of the page, such as when I found out that its .length property also accessed storage from other pages. I started out by simply writing a function to get the most recent item of local storage on the main page. By some stroke of luck, this was the exact function I needed to accomplish loading past images from the sidebar several hours later (Something the professors unfortunately didn't notice in the feedback party). Creating the sidebar was definetly the hardest part of the project as it required me to nearly double the amount of variables to keep track of (especially considering those pesky cookies) and required some amount of heavy computation whenever it needed to be updated. This is about the point where I learned that live server doesn't handle those kinds of resources as well as an actual server, because I could only get an image or two before it would crash (and god forbid I attempt to use inspect element). Don't even get me started on resizing the sidebar once a breakpoint is hit. Overall, I used over 350 lines of code to accomplish everything, and that could probably be expanded even more if I were to break statements up. In true fashion, I did all of my CSS an hour before the feedback party started, a perk of working on a web application who's primary goal is simplicity. While the feedback party was helpful to learn more about page design, it was way too close to the due date of the project, and I wans't really left with any time to implement the feedback I was given, though my page had essentially met all the requirenments already. Something that I did find out from it though was that my size field only worked on live server and not the browser because my boolean variables were too loosly defined. Didn't know that could be interpreted differently.

Errors

I've been pretty thourogh with error checking so I'm pretty sure that no more errors will be thrown, from my js file, at least. I have occasionally seen RITs server and javascript throw warnings on my program, which I'm unsure I have the ability to do anything about. In the case of user error, my program tells the user what they did wrong. It also displays to the user when the API is working, which can take a fairly long time for some images. I made sure everything validates.

Responsive

My layout switches to stack on top of eachother when close to a phone size screen. It was actually very easy to make it responsive because I had to use less CSS for a smaller screen size, and only really needed a grid for a larger one.
